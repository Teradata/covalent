When to Use Classless Mode

✅ Great For:

  • Documentation sites - Markdown content automatically styled
  • Blog posts - Focus on content, not classes
  • Prototyping - Rapid HTML without class overhead
  • Static content - Pages with simple structure
  • Accessibility - Encourages semantic HTML

⚠️ Use Utility Classes Instead For:

  • Complex UIs - Need different styles for same elements
  • Component libraries - Require fine-grained control
  • Design systems - Multiple variations of elements
  • Specific overrides - Custom styling per instance

💡 Pro Tip: Mix Both!

You can use elements.css as a base and import covalent.css for specific overrides when needed.
